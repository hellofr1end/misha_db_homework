# Техническое задание: Проектирование базы данных «Генеральный Штаб»


## Архитектура базы данных (Таблицы и связи)

Тебе необходимо создать **6 таблиц**, настроив между ними связи различного типа (1:1, 1:M, M:M) и прописать ограничения целостности данных.

### 1. Таблица альянсов (`alliances`)
Хранит информацию о крупных межгосударственных союзах.

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный ID альянса. |
| `name` | `VARCHAR(100)` | `NOT NULL`, `UNIQUE` | Название (например, 'Союзники', 'Ось', 'Коминтерн'). |
| `faction_leader` | `VARCHAR(100)` | `NOT NULL` | Страна-основатель и лидер фракции. |

---

### 2. Таблица стран (`countries`)
> **Связь «Один ко многим» (1:M) с альянсами:** В одном альянсе может быть много стран, но страна состоит только в одном альянсе (или играет без фракции).

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный ID страны. |
| `name` | `VARCHAR(100)` | `NOT NULL`, `UNIQUE` | Название страны (например, 'СССР', 'Германия'). |
| `stability` | `INTEGER` | `DEFAULT 50`, `CHECK (stability BETWEEN 0 AND 100)` | Стабильность страны в %. |
| `alliance_id` | `INTEGER` | `REFERENCES alliances(id) ON DELETE SET NULL` | Ссылка на альянс. Если альянс распадается (удаляется), страна остается, а поле получает `NULL`. |

---

### 3. Таблица Генеральных штабов (`general_staffs`)
> **Связь «Один к одному» (1:1) со странами:** У каждой страны есть ровно один Главный Штаб, координирующий её действия.

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `country_id` | `INTEGER` | `PRIMARY KEY`, `REFERENCES countries(id) ON DELETE CASCADE` | Внешний ключ, являющийся первичным. При удалении страны штаб удаляется автоматически. |
| `headquarters_city`| `VARCHAR(100)` | `NOT NULL` | Город, где расположен штаб (например, 'Москва', 'Берлин'). |
| `command_power_limit`| `INTEGER` | `DEFAULT 100`, `CHECK (command_power_limit > 0)` | Лимит очков командования для связи с армиями. |

---

### 4. Таблица генералов (`generals`)
> **Связь «Один ко многим» (1:M) со странами:** Генерал служит только одной стране, но у страны может быть много генералов.

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный ID генерала. |
| `name` | `VARCHAR(100)` | `NOT NULL` | Имя военачальника (например, 'Жуков', 'Роммель'). |
| `skill_level` | `INTEGER` | `DEFAULT 1`, `CHECK (skill_level BETWEEN 1 AND 10)` | Уровень навыка генерала. |
| `country_id` | `INTEGER` | `REFERENCES countries(id) ON DELETE CASCADE` | Ссылка на страну службы. Если страна прекращает существование, генералы удаляются. |

---

### 5. Таблица военных операций (`military_operations`)
Хранит планы военных кампаний.

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный ID операции. |
| `codename` | `VARCHAR(100)` | `NOT NULL`, `UNIQUE` | Кодовое имя операции (например, 'Барбаросса', 'Оверлорд'). |
| `difficulty` | `VARCHAR(50)` | `DEFAULT 'Normal'` | Сложность планирования. |

---

### 6. Таблица командующих операциями (`operation_commanders`)
> **Связь «Многие ко многим» (M:M):** Связующая таблица между генералами и операциями. Один генерал может участвовать в разных операциях, а в одной операции могут быть задействованы несколько генералов.

| Колонка | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| `general_id` | `INTEGER` | `REFERENCES generals(id) ON DELETE CASCADE` | Ссылка на генерала. |
| `operation_id` | `INTEGER` | `REFERENCES military_operations(id) ON DELETE CASCADE` | Ссылка на операцию. |
| `role` | `VARCHAR(100)` | `NOT NULL` | Роль в этой операции (например, 'Оборона фланга'). |

> **Важно:** Сделай связку `(general_id, operation_id)` составным первичным ключом (`PRIMARY KEY`).

---

## Практическое задание: Наполнение данными

После того как структура таблиц будет успешно создана, напиши SQL код для заполнения базы тестовыми данными.
